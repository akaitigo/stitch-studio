# ユースケース

## UC-1: 画像から図案を生成する

**アクター**: ユーザー
**前提条件**: ブラウザでアプリにアクセスしている

1. ユーザーが画像ファイル（PNG/JPEG）をアップロードする
2. システムがグリッドサイズ（幅x高さ）と最大色数の設定UIを表示する
3. ユーザーが設定を確定する
4. WASM エンジンが画像をグリッドデータに変換する（色数削減 + ピクセルマッピング）
5. グリッドエディタに変換結果が表示される

**代替フロー**:
- 3a. 画像が大きすぎる場合、エラーメッセージを表示

## UC-2: グリッド図案を手動編集する

**アクター**: ユーザー
**前提条件**: グリッドエディタが表示されている

1. ユーザーがカラーパレットから色を選択する
2. ユーザーがグリッド上のセルをクリック/ドラッグして色を塗る
3. システムがリアルタイムでグリッドを更新する

**ツール**: ペン / 塗りつぶし（フラッドフィル）/ 消しゴム

## UC-3: 糸の買い物リストを確認する

**アクター**: ユーザー
**前提条件**: グリッドに図案データがある

1. ユーザーが「糸リスト」パネルを開く
2. システムがグリッド上の各色を最寄りDMC糸番号にマッピングする
3. 各色の使用マス数から必要束数を算出する
4. DMC番号 + 色名 + 必要束数の一覧を表示する

## UC-4: 新規グリッドを作成する

**アクター**: ユーザー

1. ユーザーが「新規作成」を選択する
2. グリッドサイズ（幅x高さ）を入力する
3. 空のグリッドがエディタに表示される
4. 手動で図案を描き始める
