# ADR-0001: 画像処理エンジンに Rust/WASM を採用

## ステータス
承認

## コンテキスト
画像→グリッド変換処理（色数削減 + ピクセルマッピング）はピクセル単位の計算が必要で、パフォーマンスが重要。ブラウザ内で完結させたい。

## 決定
画像処理エンジンを Rust で実装し、wasm-pack + wasm-bindgen でWASMモジュールとしてブラウザに配信する。

## 選択肢
| 選択肢 | 長所 | 短所 |
|--------|------|------|
| A: Rust/WASM | ネイティブ並の速度、メモリ安全、型安全 | ビルドチェーン複雑、学習コスト |
| B: TypeScript (Canvas API) | ビルド単純、1言語で完結 | 大画像でパフォーマンス問題、色数削減アルゴリズムが遅い |
| C: サーバーサイド処理 | 言語選択自由 | レイテンシ、サーバーコスト、オフライン不可 |

## 結果
モノレポ構成（wasm/ + frontend/）とし、wasm-pack でビルドしたパッケージをフロントエンドから import する。CIで cargo test + wasm-pack build の両方を検証する。
